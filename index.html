Motor automático

GitHub Pages muestra la web. n8n, pasarela, base de datos y proveedor hacen la operación pesada.

1. Pedido capturado

El checkout recopila datos del cliente, productos, cantidades, envío, método de pago y total.

2. Flujo n8n

El pedido se manda a un webhook para registrar, validar, notificar y enviar datos al proveedor.

3. Seguimiento

El sistema puede enviar correo de confirmación, tracking, alertas internas y reporte diario.

Garantía y entrega

Entrega referencial: 2 a 7 días hábiles según ciudad y proveedor. Antes de vender en producción debemos agregar RUC, políticas, cambios, devoluciones, libro de reclamaciones y proveedor confirmado.
